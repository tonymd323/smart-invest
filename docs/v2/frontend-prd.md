## 一、核心价值

**痛点：** 数据采集→Bitable(5张表)→飞书查看，链条长、调试慢、无法手动触发。  
**目标：** 一站式仪表盘——触发采集 + 查看数据 + 筛选交互 + 系统监控 + 决策辅助。

---

## 二、技术栈

```
FastAPI (Python 服务器)
├── Jinja2 (模板渲染，HTML 直出)
├── HTMX (轻量 AJAX 交互，~14KB)
├── Server-Sent Events (实时日志流)
├── Plotly (图表，Python 原生)
├── Tailwind CSS CDN (样式)
└── SQLite (数据源，直读)
```

**原则：** 零构建工具、零 npm、零 webpack。全栈 Python。  
**访问地址：** http://服务器IP:8080

## 三、页面架构

### 🏠 总览 (Dashboard) — 默认首页

「一页看全局」决策信息页。

```
┌─ 今日信号 ──── 持仓状态 ──── 系统健康 ─┐
│ 超预期 2     福耀 +1.8%    Cron ✅     │
│ 扣非新高 3   海油 -0.5%    DB 4.2MB    │
│ 回调信号 1   东电 +0.7%    API 延迟 正常│
└────────────────────────────────────────┘
  📌 今日行动摘要
  📰 最新事件（最近 3 条）
  📊 本周信号胜率
```

### 📌 今日行动 — 核心决策界面

系统直接告诉用户「该做什么」。

```
📌 今日行动清单
┌──────────────────────────────────────────────┐
│ 🔥 高优先  600660 超预期+回调到位+发现池内   │
│    → 建议买入 ¥57，目标 ¥75，止损 ¥50        │
│ ⚡ 中优先  600875 扣非新高但未回调            │
│    → 加入关注，等回调至 ¥45                   │
│ 👀 观察    002232 扣非新高但市值偏小           │
│    → 等行业确认                               │
└──────────────────────────────────────────────┘
```

**排序逻辑：** 评分 × 信号强度 × 回调到位度 = 综合优先级

### 🔍 发现池

- 筛选：信号(buy/watch/hold) / 来源 / 行业 / 评分范围
- 排序：评分 / 入池时间 / 市值
- 展开：点击股票 → 弹出详情卡（基本面 + 行情 + 事件历史）
- 操作：手动入池 / 手动过期 / **记录决策**

### 📋 信号看板

- 超预期信号表
- 扣非新高信号表
- 回调买入信号表
- 筛选：日期 / 股票代码 / 信号类型 / 公告类型
- 操作：**记录决策**（已买入 / 未买入-原因 / 观望）

### 📰 事件流

- 时间线视图（默认）/ 表格视图（切换）
- 筛选：类型(增产/合同/回购/暴雷) / 情感(利好/利空/中性) / 严重程度
- 操作：标记已读 / 关联 T+N 跟踪

### 📈 T+N 跟踪

- 入池股票收益曲线图（入池日 vs 当前 vs 止盈/止损线）
- 筛选：状态(进行中/已达标/已止损/已过期)
- 统计：胜率 / 平均收益 / 最大回撤

### 💼 持仓快照（总览页子模块）

- 实时行情（腾讯 API）
- 盈亏状态 + 目标价 + 止损价
- 风险预警（油价/毛利率等）

### 📊 策略胜率

按信号类型拆分统计：

| 信号类型 | 出现次数 | 平均涨幅 | 胜率 | 平均持仓天数 |
|---------|---------|---------|------|-----------|
| 超预期 | 47 | +12.3% | 62% | 23 |
| 扣非新高 | 31 | +8.1% | 55% | 18 |
| 回调买入 | 12 | +15.7% | 75% | 35 |

### 🎯 回测

- 历史信号收益柱状图
- 按信号类型分组对比
- 导出 CSV

### ⚙️ 系统控制

- **手动触发 Pipeline**
  - 扫描模式：早盘(18h) / 盘后(4h) / 晚间(12h) / 自定义
  - 模拟时间：日期时间选择器
  - 实时进度条（SSE 日志流）
  - 结果预览 + 确认写入
- Cron 任务管理（查看/启用/禁用）
- 数据库浏览器（所有表的 raw 数据）
- 日志查看器（最近 N 条 Pipeline 日志）

## 四、决策记录功能

每条信号旁加「记录决策」按钮：

```
[ 已买入 ] [ 未买入-原因(下拉) ] [ 观望 ]
```

**未买入原因选项：** 市值太小 / 价格偏高 / 不在行业 / 不了解 / 等回调 / 其他

**效果：** 系统可计算——
- 采纳率：你采纳了 X% 的信号
- 采纳胜率 vs 未采纳胜率：信系统是对的还是错的
- 每种信号类型的 ROI

## 七、与飞书的关系

| 场景 | 前端 | 飞书 |
|------|------|------|
| 开发调试 | ✅ 主要工具 | ❌ 不用 |
| 日常查看 | ✅ 更丰富的交互 | ✅ 飞书卡片推送（被动通知） |
| 数据回填 | ✅ 手动触发 + 预览 | ❌ 不用 |
| 紧急监控 | ✅ 实时系统状态 | ✅ 飞书消息提醒 |
| 决策记录 | ✅ 唯一入口 | ❌ 不用 |
| 策略复盘 | ✅ 策略胜率看板 | ❌ 不用 |

**结论：** 前端是**决策 + 调试 + 复盘**的工具，飞书是**被动通知**的通道。两者互补。
